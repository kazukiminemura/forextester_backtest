# TradingView インジケーター

## 追加方法

1. TradingViewで対象銘柄の1時間足チャートを開きます。
2. 画面下部の「Pineエディタ」を開きます。
3. 既存コードを全選択して削除します。
4. `tamukai_pair_signal_indicator.pine`を1行目の`//@version=6`から最終行まで、
   省略せずすべて貼り付けます。
5. 「保存」→「チャートに追加」を押します。

更新版が反映されると、右上テーブルの先頭に`田向式 RR1 v2`、利確欄に
`RR 1:1 / 100%`と表示されます。以前の表示が残る場合は、チャート上の旧版を
削除してから保存済みの新版を追加し直してください。

4時間足などへ追加した場合、売買シグナルは停止し、オレンジ色の背景と
「1時間足専用」の警告を表示します。上部の時間足を`1h`へ変更してください。

## 表示

- `SET BUY` / `SET SELL`: 条件成立。レンジ外の逆指値候補を待機中
- `BUY` / `SELL`: 確定1時間足で待機価格を突破
- 黄線: 待機レンジ
- 白／緑／赤線: Entry・1:1のTP・構造SL
- `TP` / `SL`: インジケーター内の仮想到達判定

設定の`プリセット=Auto`では、JPYクロスにUSDJPY探索値、EURUSDに
`EURUSD研究`、XAUUSD（金）に`XAUUSD研究`を使います。他の銘柄はPine既定値です。
EURUSD研究設定は2023～2024年の最終評価で期待値がマイナスだったため、
テーブルへ警告を表示します。XAUUSD研究設定は2025～2026年の未使用期間で
13取引、PF 1.22でしたが、標本が少ないため研究用として警告色で表示します。

決済は全銘柄・全プリセットで「SLまでの距離＝TPまでの距離」とし、
1R到達時に100%決済します。半分利確と残りの追随は使用しません。
待機シグナル中もEntry、SL、対称位置のTPを同時に表示し、状態テーブルには
価格を数値表示します。

信号回数を増やす場合は、1時間足で`プリセット=JPY頻度優先`を選びます。
RR 1:1・1R全量決済は維持し、上位足の壁までの最低余地を1.5Rから1.0Rへ
緩めます。USDJPYの完全未使用期間（2024-09～2026-07）は12件から15件へ増え、
勝率80.00%、純損益+28,850円でした。一方、2023～2024年のホールドアウトは
赤字だったため研究用であり、Autoには適用していません。

XAUUSD研究は「買いのみ・1Rで全量利確」の厳密なRR 1:1設定です。
4本レンジ、最大1.50 ATR、4H傾き0.06 ATR、最小押し0.25 ATR、参照帯許容幅
0.40 ATR、壁まで1R以上、損切り2.0 ATR以下、待機8本を使用します。
エントリーと損切りバッファはバックテストに合わせて価格差0.40 USDへ固定し、
絶対pipsの最大損切り幅は無効にします。全期間は123取引、勝率62.60%、
PF 1.395でしたが、未使用期間は13取引だけなので研究候補です。

JPYクロス、EURUSD研究、Pine既定、Customも決済だけはRR 1:1へ統一されます。
過去に記載したJPY/EURUSD成績は別の決済条件によるため、RR 1:1変更後の成績として
使用できません。JPYクロスのRR 1:1共通値を再探索しましたが、現在設定は
2023～2024年の4通貨合算で45取引、PF 0.53、最良の代替候補もPF 0.56でした。
ホールドアウトを通過した候補がないため、売買条件は変更せず警告表示にしています。
詳細は[`../../docs/jpy_cross_rr1_optimization.md`](../../docs/jpy_cross_rr1_optimization.md)を
参照してください。

## アラート

インジケーターを追加後、TradingViewの「アラートを作成」で条件欄から次を選べます。

- `田向式 BUY setup` / `SELL setup`
- `田向式 BUY breakout` / `SELL breakout`
- `田向式 Target reached` / `Stop reached`

Pineコードを追加しただけではアラートは稼働しません。TradingView画面でアラートを
作成してください。

## 複数通貨ペアのチャンス一覧

`tamukai_pair_signal_indicator.pine`本体に、設定した10銘柄を確定済み1時間足・
4時間足で同時監視する一覧を内蔵しています。別のインジケーターを追加する必要は
ありません。設定の`通貨ペア・チャンス一覧`で表示のON/OFFと監視銘柄を変更できます。

左上の一覧は次の意味です。

- `BUY待ち`（明るい緑）: 買いの全SET条件が成立。表示Entryの上抜け待ち
- `SELL待ち`（明るい赤）: 売りの全SET条件が成立。表示Entryの下抜け待ち
- `上昇監視`（濃い緑）: 4Hは上向きだが、1Hレンジ条件は未成立
- `下降監視`（濃い赤）: 4Hは下向きだが、1Hレンジ条件は未成立
- `なし`（灰色）: 現在は方向条件なし

既定はEURUSD、GBPUSD、USDJPY、EURJPY、GBPJPY、AUDJPY、EURGBP、GBPCHF、
AUDUSD、USDCHFです。インジケーター設定の`監視銘柄`から、利用中のブローカーの
銘柄へ変更できます。`BUY待ち`・`SELL待ち`はエントリー済みではなく、既存の
同じインジケーターを対象チャートで開いてEntry・SL・TPを最終確認してください。

TradingView Pineはウォッチリスト全体を自動取得できないため、このスキャナーは
指定した10銘柄だけを監視します。また、一覧は新しいSET候補を示し、過去バーで
発生して現在も期限内の待機注文状態までは保持しません。

## 制約

このファイルは売買注文を出さない`indicator()`です。TP・SLは確定足OHLCによる
参考表示で、Forex Testerのスプレッド付き約定や同一バー内価格経路とは一致しない
場合があります。勝率・期待値の評価にはPythonバックテストを使用してください。
